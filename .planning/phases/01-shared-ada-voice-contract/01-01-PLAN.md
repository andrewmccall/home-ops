Create `cluster/apps/openclaw/instance/gateway-allow-house-voice.yaml` following the exact pattern
from `gateway-allow-internal-nginx.yaml` (per D-01):

```yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: home-ops-openclaw-allow-house-voice
  namespace: openclaw
spec:
  podSelector:
    matchLabels:
      app.kubernetes.io/instance: home-ops-openclaw
      app.kubernetes.io/name: openclaw
  policyTypes:
    - Ingress
  ingress:
    - from:
        - namespaceSelector:
            matchLabels:
              kubernetes.io/metadata.name: house
      ports:
        - port: 18790
          protocol: TCP
```

Key differences from the nginx policy:
- `namespaceSelector` uses `house` instead of `networking` (allows ALL pods in house namespace, not a specific pod — acceptable because the house namespace is a trusted internal namespace)
- NO `podSelector` under `from` (HA pods don't have stable labels from the app-template helm chart that are safe to depend on)
- Only port 18790 (gateway pod port); NOT 18794 (canvas pod port — HA doesn't need canvas)

Then add `gateway-allow-house-voice.yaml` to the `resources` list in `kustomization.yaml`, placing
it after `gateway-allow-internal-nginx.yaml` to keep NetworkPolicies grouped:

```yaml
resources:
  - openclaw-api-keys-secret.sops.yaml
  - social-api-keys.sops.yaml
  - openclaw-instance.yaml
  - allow-otel-egress.yaml
  - gateway-allow-internal-nginx.yaml
  - gateway-allow-house-voice.yaml
  - gateway-ingress.yaml
```
