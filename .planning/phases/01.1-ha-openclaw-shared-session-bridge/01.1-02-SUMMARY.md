# Phase 01.1 Plan 02: Bridge deployment and proxy ownership Summary

**Home Assistant now has a dedicated in-cluster bridge package that preserves the shared Ada session path and owns the spoofed OpenAI ingress/TLS route ahead of cutover.**

## Performance

- **Duration:** 5 min
- **Started:** 2026-04-13T09:05:44Z
- **Completed:** 2026-04-13T09:10:30Z
- **Tasks:** 2
- **Files modified:** 8

## Accomplishments

- Added a deployable `cluster/apps/openclaw/bridge/` package with a mounted Node runtime, Service, secret wiring, and deterministic degraded-mode toggle support.
- Moved the spoof `api.openai.com` ingress and TLS manifests under bridge ownership and repointed ingress traffic at the new bridge service.
- Wired the top-level OpenClaw kustomization to include the bridge while removing the old spoof ingress/TLS resources from the LiteLLM package.

## Task Commits

Each task was committed atomically:

1. **Task 1: Create the HA-facing passthrough bridge with deterministic continuity injection** - `756ee791` (feat)
2. **Task 2: Move api.openai.com ingress and TLS ownership from LiteLLM to the bridge without removing LiteLLM yet** - `eaef46ed` (feat)

## Files Created/Modified

- `cluster/apps/openclaw/bridge/bridge-runtime-configmap.yaml` - embedded Node bridge runtime for auth, passthrough, continuity headers, streaming, and degraded-mode signaling.
- `cluster/apps/openclaw/bridge/bridge-secret.sops.yaml` - encrypted bridge-facing API key, upstream gateway token, shared session key, and degraded-mode toggle.
- `cluster/apps/openclaw/bridge/bridge.yaml` - bridge Deployment and Service wiring.
- `cluster/apps/openclaw/bridge/kustomization.yaml` - bridge package resource entrypoint.
- `cluster/apps/openclaw/bridge/openai-proxy-ingress.yaml` - bridge-owned internal spoof ingress for `api.openai.com`.
- `cluster/apps/openclaw/bridge/openai-proxy-tls-secret.yaml` - bridge-owned TLS secret for the spoof path.
- `cluster/apps/openclaw/kustomization.yaml` - includes the bridge package in the active OpenClaw deployment.
- `cluster/apps/openclaw/litellm/kustomization.yaml` - drops the old ingress/TLS resources so LiteLLM no longer owns the spoof path.

## Decisions Made

- Kept the bridge runtime as a mounted ConfigMap script so the HA-facing adapter remains easy to inspect and update without introducing a separate image build.
- Made degraded mode explicit in multiple layers instead of relying on silent fallback, so operators and users can tell when shared continuity is unavailable.

## Issues Encountered
