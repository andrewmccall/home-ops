<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Home Assistant → bridge | HA sends bearer-authenticated requests to a public-looking OpenAI-compatible endpoint |
| bridge → OpenClaw | Bridge upgrades trust, injects identity headers, and uses the upstream gateway token |
| ingress/TLS → bridge service | Internal spoof path terminates TLS before forwarding to the bridge |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-01.1-04 | S | Bridge auth surface | mitigate | Require a dedicated HA-facing bridge key in `bridge-secret.sops.yaml`; never expose the upstream OpenClaw token to HA |
| T-01.1-05 | T | Continuity metadata injection | mitigate | Inject fixed session/channel headers in the bridge only; make degraded fallback explicit per D-06 |
| T-01.1-06 | D | Ingress/TLS cutover | mitigate | Move ingress/TLS ownership before LiteLLM removal so the HA spoof path stays live during cutover |
</threat_model>
